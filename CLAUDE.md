# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is a product specification repository for the **Operator Analytics Dashboard** — a white-label admin panel module for BlaBlaCar Daily's B2B/B2G partnerships. It contains PRD documents, not source code.

## Key Context

- The main document is `prd-draft.md` (v0.1, In Discovery status)
- The dashboard module covers five analytics areas: Adoption, Ride Activity, Matching/Funnel, Subsidy/Budget, and Impact
- Target users are Operator Admins (mobility managers, HR leads, sustainability leads)
- The spec follows MoSCoW prioritization and includes RICE scoring
- All metrics must be modeled as reusable configuration — no bespoke per-operator engineering

## Working with This Repository

- When editing the PRD, preserve the existing structure: Metadata, Stakeholders, Problem Statement, Strategic Goal, RICE, Scope, Personas, User Stories, Functional Requirements, Acceptance Criteria, KPI Set, Success Metrics, Out of Scope, Open Questions, Rollout Approach
- Items marked `[Hypothesis]` are unvalidated assumptions requiring discovery
- The PRD is a living document — maintain the version number in Metadata when making significant changes
